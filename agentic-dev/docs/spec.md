# Spec

Use this file only when the project has enough moving parts to need a clearer blueprint. Or when you want to start with a spec before coding. This is where you can write down the key components, data, rules, and the first slice of behaviour to implement. You can also use it to explore different design options and make decisions about the architecture of your project before you start coding. This can help you avoid getting stuck in a particular design or implementation approach that might not be the best one for your project.

The headings below may or may not be useful depending on your project and the type of spec you want to write. If you use an MCP like juxt/allium or github/spec-kit, it will use its own spec format.
